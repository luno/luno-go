# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Go SDK for the [Luno API](https://www.luno.com/api). Module: `github.com/luno/luno-go`. Public; consumed by external developers via `go get`. Go version is pinned in `go.mod` (currently 1.26).

## Commands

```bash
go vet ./...            # CI runs this
go test -v ./...        # CI runs this
go test -run TestFoo ./...                 # single test by name
go test -run TestFoo ./streaming/...       # single test in one package
go test -race ./...
```

There is no separate lint config beyond `go vet`. SonarCloud runs in CI (`.github/workflows/sonar.yml`).

The `_examples/` directory is excluded from the build by Go's leading-underscore convention - run examples with `go run ./_examples/readonly` etc.

## Architecture

Three packages, one root + two subpackages:

- **`luno`** (root) - REST client. The HTTP client, auth, and rate limiting live in `luno.go`. All request/response types and method bindings are in `api.go`.
- **`luno/streaming`** - separate WebSocket client for the order-book streaming API. Maintains a local order book by applying create/delete/trade updates against a snapshot. Reconnects with backoff.
- **`luno/decimal`** - fixed-point decimal type backed by `math/big.Int`. Used throughout for prices, volumes, balances. Do not substitute `float64`.

### `api.go` is generated - do not hand-edit

`api.go` carries `// Code generated by sdkgen` and is regenerated from the upstream Luno API spec (see commit `eb5245a "Regenerate SDK from latest Luno spec"`). Changes to method signatures, request/response structs, or endpoint paths belong in the generator, not here. If you must patch it locally, expect the next regeneration to revert your change. Hand-written extensions (helpers, custom types, the streaming client) live in other files.

### Request marshalling

`Client.do` in `luno.go` serialises request structs into `url.Values` via the reflection-based `makeURLValues` in `util.go`. Fields are picked up by their `url:"..."` struct tag. To plug in a non-trivial type, implement `QueryValuer` (`QueryValue() string`) - this is how `luno.Time` and `decimal.Decimal` opt into query encoding. Path parameters are handled by the `{id}` placeholder convention: a field tagged `url:"id"` is substituted into the URL and then dropped from the query/body.

### Custom JSON types

- `luno.Time` (in `time.go`) - the Luno API returns timestamps as either millisecond epoch integers or RFC3339 strings depending on the endpoint. `UnmarshalJSON` accepts both; `MarshalJSON` emits RFC3339; `QueryValue` emits ms-epoch. Prefer this over `time.Time` for any API field.
- `decimal.Decimal` - stringly-encoded in JSON. Constructed via `decimal.New`, `decimal.NewFromString`, `decimal.NewFromFloat64`. Use `Cmp` for comparison, not `==`.

### Client construction

`luno.NewClient()` returns a client with a default `*http.Client` (10s timeout) and a `golang.org/x/time/rate` limiter sized to the documented 300 req/min. Auth is set separately via `SetAuth`. The limiter, HTTP client, timeout, base URL, debug logging, and User-Agent suffix all have `Set*` methods. `SetBaseURL` is for internal/test use.

### Versioning

`version.go` contains `const Version` which is baked into the User-Agent header. Note: it has drifted from the git tags in the past, so check the latest tag before assuming it reflects the released version.

## Conventions specific to this repo

- Tests use `github.com/stretchr/testify`.
- Public types must keep JSON tags stable - this SDK is consumed downstream.
- Don't introduce dependencies casually; the dependency tree is small on purpose (`gorilla/websocket`, `stretchr/testify`, `golang.org/x/time`).
